▪️サービス概要
高校野球が好きな人が集まるサービス。

◾️このサービスへの思い、作りたい理由
高校までずっと野球をしてて、野球を見るのが好きなので、一緒に見に行ったり、話したりできる人が少ないので、集めれる場を作れればもっと楽しくなるかなと思いました。

▪️ユーザー層について
まずは高校野球が好きな人。おいおいは野球好き全般にしていけたらと思っています。

▪️サービスの利用イメージ
投稿機能を作り、意見や情報交換日常などを発信できるサービス。あとは、高校野球のトーナメント表欄を作り、試合日程などを確認できる。

◾️ユーザーの獲得について
まずは野球が好きな人が集まるので、スムーズに会話できると思う。投稿機能を使わなくてもトーナメント表で試合日程や勝ち負け、試合時間どこで試合するかなどをすぐ確認できる。

◾️サービスの差別化ポイント、推しポイント
高校野球に特化して、試合日程や勝ち負けをすぐわかりやすく、野球好きと繋がりやすいと思っています。

◾️機能候補
ユーザー登録機能。post投稿機能(写真、動画、本文)。投稿いいね機能。投稿検索機能。投稿にコメント機能。高校野球地区大会の対戦カード、勝敗、点差、球場情報が見れるトーナメント表機能。
本リリース　個人チャット機能、トーナメント表機能の管理画面(csvファイルの更新ボタン、試合の対戦カード、勝敗、球場情報)

▪️機能の実装方針予定
トーナメント表欄に球場情報も入れればと思っており、google mapsと連携したい。

▪️使用技術
開発環境:Docker、サーバサイド: Ruby on Rails Ruby、フロントエンド: HotWire、Webアプリケーションサーバ: Heroku、ファイルサーバ: AWS S3、セッションサーバ: Redis、データベースサーバ: PostgreSQL
